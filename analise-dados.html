Foco CESPE: A banca cobra exaustivamente a hierarquia DIKW (Dado → Informação → Conhecimento → Inteligência), a distinção entre dados estruturados e não estruturados, os princípios de dados abertos, o ciclo de vida dos dados (coleta, tratamento, armazenamento, integração, recuperação), os tipos de análise (descritiva, exploratória, explanatória, preditiva, prescritiva), funções do Excel (Power Query, tabelas dinâmicas, fórmulas lógicas/financeiras/texto, formatação condicional, gráficos, macros/VBA), e conceitos estatísticos (variáveis, distribuição de frequências, medidas de posição/dispersão, correlação, box-plot, outliers, distribuições binomial/normal, inferência estatística, intervalos de confiança, testes de hipóteses). 1. HIERARQUIA DIKW: DADO, INFORMAÇÃO, CONHECIMENTO E INTELIGÊNCIA

📘 Conceito e Evolução do Valor dos Dados

A hierarquia DIKW (Data, Information, Knowledge, Wisdom/Intelligence) representa a transformação progressiva de elementos brutos em insights acionáveis para tomada de decisão.

🧠 Inteligência (Wisdom)
💡 Conhecimento (Knowledge)
📋 Informação (Information)
🔢 Dado (Data)

🔹 Definições e Características

NívelDescriçãoExemplo Prático (Contexto TCDF)Características Principais
DadoRegistro bruto, simbólico, sem contexto ou significado intrínseco. Representa fatos isolados."150", "2024", "R$ 5.000", "Servidor X"• Atomicidade
• Neutralidade
• Requer interpretação
• Pode ser quantitativo ou qualitativo
InformaçãoDados processados, organizados e contextualizados, que respondem a perguntas básicas (quem, o quê, quando, onde)."Servidor X recebeu R$ 5.000 em janeiro de 2024"• Contextualização
• Relevância
• Responde a perguntas descritivas
• Base para análise
ConhecimentoInformação combinada com experiência, regras e entendimento, permitindo responder "como" e "por quê"."O salário de R$ 5.000 do Servidor X está dentro da faixa salarial do cargo Y, conforme a tabela de vencimentos do TCDF"• Aplicação prática
• Baseado em experiência
• Permite tomada de decisão operacional
• Transferível entre pessoas
InteligênciaConhecimento aplicado estrategicamente para prever tendências, identificar padrões complexos e orientar decisões de alto impacto."A análise histórica indica que aumentos salariais acima de 10% em cargos de nível Z geram desequilíbrio na folha; recomenda-se revisão da política de progressão"• Visão estratégica
• Capacidade preditiva
• Geração de valor organizacional
• Suporte a decisões complexas
✅ Exemplo Prático CESPE

Cenário: Auditoria de despesas com diárias no TCDF.

• Dado: "R$ 1.200", "Brasília", "05/03/2024", "Servidor A"
• Informação: "Servidor A recebeu diária de R$ 1.200 para missão em Brasília em 05/03/2024"
• Conhecimento: "O valor de R$ 1.200 está acima do limite de R$ 800 para diárias nacionais, conforme Portaria TCDF nº XX/2023"
• Inteligência: "Padrão recorrente de diárias acima do limite no setor X indica necessidade de revisão dos controles internos e capacitação dos gestores"

Aplicação: O auditor transforma dados brutos em inteligência acionável para melhoria da gestão.

🧠 MACETE: "D-I-K-I" → Dado (bruto), Informação (contexto), Conhecimento (como/porquê), Inteligência (estratégia/predição). Memorize a evolução do valor!

3. DADOS ABERTOS

📘 Conceito e Princípios

Dados abertos são dados que podem ser livremente utilizados, reutilizados e redistribuídos por qualquer pessoa, sujeitos apenas, no máximo, à exigência de atribuição da fonte e compartilhamento pela mesma licença.

🔹 Princípios dos Dados Abertos (Open Data Charter)

  1. Dados abertos por padrão: A abertura deve ser a regra; o sigilo, a exceção justificada.
  2. Qualidade e quantidade: Dados devem ser completos, primários, atualizados, acessíveis e machine-readable.
  3. Utilizáveis por todos: Sem discriminação de usuários ou finalidades.
  4. Para melhorar a governança: Transparência, accountability e participação social.
  5. Para inovação: Fomento ao desenvolvimento econômico e social.

4. CICLO DE VIDA DOS DADOS: COLETA, TRATAMENTO, ARMAZENAMENTO, INTEGRAÇÃO E RECUPERAÇÃO

📘 Etapas do Gerenciamento de Dados

O ciclo de vida dos dados compreende as fases pelas quais os dados passam, desde sua geração até o descarte, garantindo qualidade, segurança e valor ao longo do tempo.

🔹 Detalhamento das Etapas

  1. 1Coleta (Aquisição)
    • Objetivo: Capturar dados relevantes de fontes internas e externas.
    • Técnicas: Formulários, sensores, APIs, web scraping, importação de arquivos, integração de sistemas.
    • Desafios: Qualidade na fonte, consentimento (LGPD), volume, variedade.
    • Exemplo TCDF: Importação automática de empenhos do SIAFEM via API; coleta de pareceres via formulário eletrônico.
  2. 2Tratamento (Preparação/Limpeza)
    • Objetivo: Transformar dados brutos em dados analiticamente úteis.
    • Atividades:
      • Limpeza: remoção de duplicatas, correção de erros, tratamento de missing values
      • Padronização: formatos de data, moeda, codificação de categorias
      • Enriquecimento: derivação de novas variáveis, junção com fontes externas
      • Validação: regras de negócio, consistência lógica
    • Ferramentas: Excel (Power Query), Python (pandas), R, ETL tools (Talend, Informatica).
    • Exemplo TCDF: Padronizar códigos de órgãos; tratar valores nulos em campos obrigatórios; validar se soma de parcelas = valor total do contrato.
  3. 3Armazenamento
    • Objetivo: Guardar dados de forma segura, escalável e acessível.
    • Tipos de repositórios:
      • Bancos relacionais: Dados estruturados, transacionais (SQL Server, PostgreSQL)
      • Data warehouses: Dados históricos para análise (modelagem dimensional)
      • Data lakes: Dados brutos, estruturados e não estruturados (Hadoop, cloud storage)
      • Bancos NoSQL: Dados semiestruturados, alta escalabilidade (MongoDB, Cassandra)
    • Considerações: Backup, recuperação de desastres, governança, conformidade (LGPD).
  4. 4Integração
    • Objetivo: Combinar dados de múltiplas fontes para visão unificada.
    • Desafios: Heterogeneidade de formatos, conflitos semânticos, qualidade inconsistente.
    • Técnicas:
      • ETL (Extract, Transform, Load): Extração, transformação e carregamento em batch
      • ELT (Extract, Load, Transform): Carregamento primeiro, transformação depois (cloud-native)
      • APIs e web services: Integração em tempo real
      • Master Data Management (MDM): Gestão de dados mestres (ex: cadastro único de servidores)
    • Exemplo TCDF: Integrar dados do SIAFEM (financeiro), SIGAD (processos) e RH (pessoas) para auditoria transversal.
  5. 5Recuperação (Acesso e Disseminação)
    • Objetivo: Disponibilizar dados para análise, relatórios e tomada de decisão.
    • Mecanismos:
      • Consultas SQL e linguagens de consulta
      • Ferramentas de BI (Power BI, Tableau, Qlik)
      • APIs para consumo externo
      • Portais de dados abertos
    • Controles: Autenticação, autorização, auditoria de acesso, máscara de dados sensíveis.

⚠️ PEGADINHA CLÁSSICA DO CESPE

O CESPE frequentemente afirma que "a etapa de tratamento de dados ocorre apenas após o armazenamento" → ERRADO. Na prática moderna (especialmente com ELT e ferramentas como Power Query), o tratamento pode ocorrer durante a coleta (ex: validação na entrada) ou após o carregamento no data lake. O ciclo é iterativo, não estritamente linear.

A representação dos dados define como eles são codificados para armazenamento e processamento, impactando a eficiência das análises e a precisão dos resultados.

🔹 Boas Práticas de Representação

  • Consistência: Usar o mesmo formato para o mesmo tipo de dado em todo o sistema (ex: sempre "YYYY-MM-DD" para datas).
  • Documentação: Manter dicionário de dados com descrição, tipo, domínio e regras de cada campo.
  • Normalização: Para dados textuais: padronizar maiúsculas/minúsculas, remover espaços extras, tratar acentos.
  • Codificação de categorias: Usar códigos curtos e estáveis (ex: "PREG" para pregão) em vez de descrições longas.
  • Tratamento de missing values: Distinguir entre "não se aplica", "não informado" e "valor nulo"; documentar a convenção adotada.
  • Metadados: Incluir informações sobre fonte, data de coleta, responsável e qualidade dos dados.
✅ Exemplo Prático CESPE

Cenário: Representação de valores monetários em planilha de auditoria.

Representação inadequada:
• "R$ 1.234,56" como texto → impede cálculos
• "1234.56" sem indicação de moeda → ambiguidade
• "1,234.56" (formato US) em sistema BR → erro de interpretação

Representação adequada:
• Tipo: decimal(15,2)
• Valor: 1234.56 (ponto como separador decimal)
• Metadado: moeda = "BRL", precisão = 2 casas
• Formatação de exibição: "R$ 1.234,56" (apenas para visualização)

Benefício: Cálculos precisos, integração com sistemas financeiros, conformidade com padrões contábeis.

6. TIPOS DE ANÁLISE DE DADOS

📘 Classificação por Objetivo e Complexidade

As análises de dados podem ser categorizadas conforme o nível de insight gerado e a complexidade das técnicas empregadas, formando um continuum de valor analítico.

🔹 Os Cinco Tipos de Análise (Gartner Analytics Ascend Model)

Tipo de AnálisePergunta CentralTécnicas TípicasExemplo no TCDFComplexidade
Descritiva"O que aconteceu?"Estatística descritiva, agregações, relatórios, dashboards"Qual o total gasto com diárias por órgão em 2024?"Baixa

• Foco: Resumo do passado
• Saída: Relatórios estáticos, KPIs
• Ferramentas: Excel, SQL, BI básico
• Valor: Monitoramento operacional
Exploratória:
• Foco: Descoberta de insights
• Saída: Hipóteses, visualizações
• Ferramentas: Python/R, Tableau, Power BI avançado
• Valor: Identificação de oportunidades/risco
Explanatória:
• Foco: Compreensão causal
• Saída: Modelos explicativos, fatores de influência
• Ferramentas: Estatística inferencial, regressão
• Valor: Fundamentação de decisões corretivas
Preditiva:
• Foco: Antecipação de eventos

Prescritiva:
• Foco: Otimização de ações
• Saída: Recomendações acionáveis, planos

• Valor: Maximização de resultados estratégicos
⚠️ PEGADINHA CLÁSSICA DO CESPE

O CESPE frequentemente afirma que "análise preditiva e prescritiva são sinônimos" → ERRADO. A análise preditiva responde "o que provavelmente acontecerá" (foco em previsão). A análise prescritiva vai além, respondendo "o que devemos fazer" (foco em recomendação de ações). A prescritiva geralmente incorpora a preditiva como insumo.

7. TOMADA DE DECISÃO BASEADA EM DADOS (Data-Driven Decision Making)

📘 Conceito e Pilares

Tomada de decisão baseada em dados é o processo sistemático de coletar, analisar e interpretar dados para fundamentar escolhas estratégicas, táticas e operacionais, reduzindo a dependência de intuição ou viés cognitivo.

🔹 Pilares da Decisão Data-Driven

  1. Qualidade dos dados: Dados precisos, completos, atualizados e relevantes são pré-requisito para decisões confiáveis.
  2. Contexto organizacional: Dados devem ser interpretados à luz dos objetivos, restrições e cultura da organização.
  3. Competência analítica: Capacidade da equipe de transformar dados em insights acionáveis.
  4. Cultura data-driven: Valorização da evidência empírica sobre opinião ou hierarquia.
  5. Ética e conformidade: Uso responsável dos dados, respeitando privacidade (LGPD) e transparência.

🔹 Processo de Decisão Baseada em Dados

1Definir o problema decisório: Qual decisão precisa ser tomada? Quais são os critérios de sucesso?
2Identificar dados relevantes: Quais dados internos/externos podem informar a decisão?
3Coletar e preparar dados: Garantir qualidade, integridade e representatividade dos dados.
4Analisar e modelar: Aplicar técnicas estatísticas ou de machine learning para extrair padrões e insights.
5Interpretar resultados: Traduzir achados analíticos em implicações práticas para a decisão.
6Comunicar e decidir: Apresentar evidências de forma clara aos decisores; tomar a decisão com base nas evidências.
7Monitorar e ajustar: Acompanhar os resultados da decisão; usar feedback para refinar modelos futuros.

🔹 Armadilhas Comuns e Como Evitá-las

ArmadilhaDescriçãoEstratégia de Mitigação
Correlação ≠ CausalidadeAssumir que duas variáveis correlacionadas têm relação causalRealizar testes de causalidade; considerar variáveis de confusão; usar experimentos controlados quando possível
Vieses de confirmaçãoBuscar apenas dados que confirmem hipóteses pré-existentesDefinir hipóteses nulas; buscar ativamente evidências contrárias; envolver múltiplas perspectivas na análise
OverfittingModelo muito complexo que se ajusta ao ruído dos dados de treino, perdendo capacidade de generalizaçãoValidação cruzada; separação treino/teste; regularização; simplicidade (princípio da parcimônia)
Dados desatualizadosTomar decisões com base em dados que não refletem a realidade atualEstabelecer SLAs de atualização; monitorar drift de dados; alertas para anomalias temporais
Falta de contextoInterpretar números isoladamente, sem considerar fatores qualitativos ou externosCombinar análise quantitativa com pesquisa qualitativa; envolver especialistas de domínio na interpretação
✅ Exemplo Prático CESPE

Cenário: Decisão sobre alocação de recursos para auditorias no TCDF.

Abordagem não data-driven: "Sempre auditamos os órgãos com maior orçamento."

Abordagem data-driven:
1. Problema: Maximizar detecção de irregularidades com recursos limitados.
2. Dados: Histórico de auditorias (órgão, valor auditado, irregularidades encontradas, tipo de gasto).
3. Análise: Modelo de regressão identifica que "variação percentual de despesas no trimestre" e "número de fornecedores novos" são preditores significativos de irregularidades.
4. Insight: Órgãos com crescimento abrupto de despesas + alta rotatividade de fornecedores têm 3x mais chance de irregularidades.
5. Decisão: Alocar 70% dos recursos para auditorias em órgãos com esse perfil de risco.
6. Monitoramento: Acompanhar taxa de detecção pós-alocação; ajustar modelo trimestralmente.

Resultado: Aumento de 40% na detecção de irregularidades com os mesmos recursos.

8. UTILIZAÇÃO DE EXCEL PARA ANÁLISE DE DADOS (MICROSOFT 365)

📘 Excel como Ferramenta de Análise

O Microsoft Excel (versão 365) evoluiu de planilha eletrônica para plataforma de análise de dados, integrando funcionalidades de ETL, modelagem, visualização e automação.

🔹 8.1 Importação e Manipulação de Dados via Power Query

Power Query é a ferramenta de ETL (Extract, Transform, Load) nativa do Excel, permitindo conectar, transformar e combinar dados de múltiplas fontes sem fórmulas complexas.

🔸 Fontes de Dados Suportadas

  • Arquivos: Excel, CSV, TXT, XML, JSON, PDF (tabelas)
  • Bancos de dados: SQL Server, Access, Oracle, MySQL, PostgreSQL
  • Serviços online: SharePoint, Dynamics 365, Salesforce, APIs REST
  • Outros: Web scraping, pastas, OData, ODBC

🔹 8.2 Validação de Dados

Validação de dados no Excel permite restringir o tipo ou intervalo de valores que podem ser inseridos em uma célula, prevenindo erros na entrada de dados.

🔸 Tipos de Validação

  • Lista: Restringe entrada a valores de uma lista predefinida (dropdown).
  • Número inteiro/decimal: Define intervalo mínimo/máximo, ou comparação com outra célula.
  • Data: Restringe a um período específico (ex: datas do exercício atual).
  • Comprimento do texto: Limita número de caracteres (ex: CPF com 11 dígitos).
  • Personalizada: Usa fórmula para validação complexa (ex: validar formato de CNPJ).
✅ Exemplo Prático: Validação de CPF no Excel

Objetivo: Garantir que a coluna "CPF" receba apenas valores com 11 dígitos numéricos.

Passos:
1. Selecionar a coluna "CPF"
2. Dados → Validação de Dados
3. Permitir: "Personalizado"
4. Fórmula: =E(NÚM.CARACT(A2)=11;É.NÚM(A2))
5. Mensagem de erro: "CPF deve ter exatamente 11 dígitos numéricos"

Resultado: Tentativas de inserir "123.456.789-00" ou "12345" serão bloqueadas, mantendo a integridade dos dados.

🔹 8.4 Consolidação de Dados

Consolidação permite combinar dados de múltiplas planilhas ou pastas de trabalho em um único resumo, usando funções de agregação (soma, média, contagem, etc.).

🔸 Métodos de Consolidação

  1. Por posição: Combina dados com base na localização idêntica nas planilhas de origem (ex: mesma célula A1 em 12 planilhas mensais).
  2. Por categoria: Combina dados com base em rótulos de linha/coluna (ex: consolidar despesas por "órgão", mesmo que a ordem das linhas varie entre planilhas).
  3. Com vínculos: Cria fórmulas que referenciam as células de origem, atualizando automaticamente quando os dados fonte mudam.
Exemplo de consolidação por categoria (função SOMA.SE):
=SOMA.SE('Planilha1:Planilha12'!A:A; "Órgão X"; 'Planilha1:Planilha12'!B:B)

Exemplo com Power Query (mais robusto):
1. Importar todas as planilhas mensais
2. Anexar consultas (union)
3. Agrupar por "Órgão" → Soma de "Valor"

🔹 8.5 Tabelas Dinâmicas (PivotTables)

Tabelas dinâmicas são a ferramenta mais poderosa do Excel para análise exploratória, permitindo resumir, cruzar e segmentar grandes volumes de dados com arrastar-e-soltar.

🔸 Componentes de uma Tabela Dinâmica

  • Campos: Colunas da tabela fonte que podem ser usadas na análise.
  • Áreas:
    • Filtros: Restringe os dados exibidos (ex: apenas exercício 2024)
    • Linhas: Define as categorias que aparecem nas linhas (ex: órgãos)
    • Colunas: Define as categorias que aparecem nas colunas (ex: meses)
    • Valores: Define as métricas a serem calculadas (ex: soma de valores, contagem de registros)
  • Configurações de campo de valor: Define a função de agregação (soma, média, contagem, máximo, etc.) e formatação.

🔸 Técnicas Avançadas

  • Agrupamento: Agrupar datas por mês/trimestre/ano; agrupar valores numéricos em faixas (ex: faixas salariais).
  • Campos calculados: Criar novas métricas com fórmulas (ex: "Ticket médio" = Soma de valor / Contagem de registros).
  • Segmentação de dados (Slicers): Filtros visuais interativos para dashboards.
  • Gráficos dinâmicos: Gráficos vinculados à tabela dinâmica, atualizando automaticamente com os filtros.
  • Atualização automática: Configurar para atualizar ao abrir o arquivo ou em intervalo definido.
✅ Exemplo Prático: Auditoria de Despesas com Tabela Dinâmica

Objetivo: Identificar órgãos com despesas atípicas por tipo de gasto.

Passos:
1. Selecionar a tabela de despesas
2. Inserir → Tabela Dinâmica
3. Arrastar:
  • Linhas: "Órgão"
  • Colunas: "Tipo de Despesa"
  • Valores: "Valor" (Soma)
4. Aplicar filtro de valor: mostrar apenas órgãos com total > R$ 1 milhão
5. Adicionar formatação condicional: destacar células acima de 2 desvios-padrão da média por tipo
6. Inserir segmentação por "Mês" para análise temporal interativa

Resultado: Dashboard interativo que permite identificar rapidamente órgãos com padrões de gasto fora do esperado.

🔹 8.7 Formatação Condicional

Formatação condicional aplica formatação visual (cores, ícones, barras de dados) a células com base em regras, facilitando a identificação de padrões e anomalias.

🔸 Tipos de Regras

  • Destacar regras de células: Formatar se valor é maior/menor/igual a um limite, ou está entre valores, ou contém texto específico.
  • Regras de primeiros/últimos: Destacar os N maiores/menores valores, ou valores acima/abaixo da média.
  • Barras de dados / Escalas de cor / Conjuntos de ícones: Visualização in-cell para comparação rápida.
  • Fórmulas personalizadas: Usar qualquer fórmula lógica para definir a condição (ex: formatar se valor > média + 2*desvio-padrão).
✅ Exemplo Prático: Identificar Outliers com Formatação Condicional

Objetivo: Destacar despesas com valor atípico (acima de 2 desvios-padrão da média).

2. Página Inicial → Formatação Condicional → Nova Regra

🔹 9.3 Estatística Descritiva e Análise Exploratória de Dados

📊 Objetivos da Estatística Descritiva

Resumir e descrever as características principais de um conjunto de dados, sem fazer inferências sobre uma população maior.

🔸 9.3.1 Distribuição de Frequências

Já detalhada acima. Importante para identificar a forma da distribuição (simétrica, assimétrica, bimodal) antes de escolher medidas de posição/dispersão adequadas.

🔸 9.3.2 Medidas de Posição

Média Aritmética (x̄): x̄ = Σxi / n

Mediana (Md): Valor que divide a amostra ordenada ao meio
• n ímpar: posição (n+1)/2
• n par: média dos valores nas posições n/2 e n/2+1

Moda (Mo): Valor com maior frequência

Quartis (Q1, Q2=Mediana, Q3): Dividem a amostra em 4 partes iguais
• Q1 (25º percentil): 25% dos dados ≤ Q1
• Q3 (75º percentil): 75% dos dados ≤ Q3

Percentis (Pk): Valor abaixo do qual caem k% dos dados
MedidaVantagensLimitaçõesQuando Preferir
Média• Utiliza todos os dados
• Base para inferência estatística
• Sensível a mudanças em qualquer valor		• Muito sensível a outliers
• Pode não representar bem distribuições assimétricas		Distribuições simétricas, sem outliers extremos
Mediana• Robusta a outliers
• Representa bem o "centro" de distribuições assimétricas
• Sempre existe e é única		• Não utiliza todos os dados
• Menos eficiente estatisticamente que a média		Distribuições assimétricas, presença de outliers, dados ordinais
Moda• Aplicável a dados qualitativos

• Pode haver múltiplas modas (bimodal, multimodal)		• Pode não existir ou não ser única
• Não utiliza magnitude dos valores		Dados categóricos, identificação de valores mais frequentes

🔸 9.3.3 Medidas de Dispersão

Amplitude: A = Xmáx - Xmín

Variância Amostral (s²): s² = Σ(xi - x̄)² / (n - 1)

Desvio-Padrão Amostral (s): s = √s²

Coeficiente de Variação (CV): CV = (s / x̄) × 100%

Intervalo Interquartil (IQR): IQR = Q3 - Q1
MedidaInterpretaçãoUnidadeAplicação Prática
AmplitudeDiferença entre maior e menor valorMesma dos dadosVisão rápida da extensão dos dados; sensível a outliers
VariânciaMédia dos quadrados dos desvios em relação à médiaUnidade² dos dadosBase para inferência; difícil interpretação direta devido à unidade ao quadrado
Desvio-PadrãoDesvio médio dos dados em relação à médiaMesma dos dadosMedida mais intuitiva de dispersão; base para regra empírica (68-95-99.7)
CVDesvio-padrão como porcentagem da médiaAdimensional (%)Comparar dispersão entre conjuntos com unidades ou médias diferentes
IQRAmplitude dos 50% centrais dos dadosMesma dos dadosMedida robusta de dispersão; base para identificação de outliers em box-plot

🔸 9.3.4 Correlação

Mede a força e direção da relação linear entre duas variáveis quantitativas.

Coeficiente de Correlação de Pearson (r):
r = Σ[(xi - x̄)(yi - ȳ)] / √[Σ(xi - x̄)² × Σ(yi - ȳ)²]

Interpretação:
• -1 ≤ r ≤ 1
• r ≈ 1: correlação positiva forte
• r ≈ -1: correlação negativa forte
• r ≈ 0: ausência de correlação linear

Atenção: Correlação ≠ causalidade!

🔸 9.3.5 Histogramas e Curvas de Frequência

  • Histograma: Gráfico de barras que mostra a distribuição de frequências de uma variável contínua, com barras adjacentes representando intervalos (bins) de valores.
  • Curva de frequência: Linha suave que aproxima a forma do histograma, útil para visualizar a forma da distribuição (normal, assimétrica, bimodal).
  • Dica: Escolher número adequado de bins (regra de Sturges: k ≈ 1 + 3.322 × log10(n)) para equilibrar detalhe e clareza.

🔸 9.3.6 Diagrama Box-Plot

Representação gráfica que resume a distribuição de uma variável quantitativa através de cinco números-resumo: mínimo, Q1, mediana, Q3, máximo, além de outliers.

Limites para outliers:
Limite inferior = Q1 - 1,5 × IQR
Limite superior = Q3 + 1,5 × IQR

Valores fora desses limites são plotados como pontos individuais (outliers).

🔸 9.3.7 Avaliação de Outliers

Outliers são observações que se afastam significativamente do padrão geral dos dados.

🔹 Métodos de Detecção
  1. Regra do IQR (box-plot): Valores < Q1 - 1,5×IQR ou > Q3 + 1,5×IQR.
  2. Regra do desvio-padrão: Valores com |z-score| > 3 (ou > 2 para amostras pequenas), onde z = (x - x̄)/s.
  3. Visual: Gráficos de dispersão, box-plot, histograma.
  4. Modelagem: Resíduos de regressão, algoritmos de detecção (Isolation Forest, DBSCAN).
🔹 Abordagem para Tratamento
  • Investigar causa: Erro de digitação? Evento legítimo mas raro? Mudança estrutural?
  • Decidir ação:
    • Corrigir: se for erro de entrada
    • Manter: se for observação válida e relevante para a análise
    • Transformar: aplicar log, raiz quadrada para reduzir assimetria
    • Segmentar: analisar outliers separadamente
    • Excluir: apenas se justificado e documentado (risco de viés)
⚠️ PEGADINHA CLÁSSICA DO CESPE

O CESPE frequentemente afirma que "outliers devem sempre ser removidos da análise" → ERRADO. A remoção automática de outliers pode introduzir viés e mascarar fenômenos importantes. A decisão deve ser baseada em investigação da causa e justificativa analítica, documentada no relatório.

🔹 9.4 Análise de Dados Categorizados

Técnicas para analisar relações entre variáveis qualitativas ou entre qualitativas e quantitativas.

🔸 Tabela de Contingência

Tabela de dupla entrada que mostra a distribuição conjunta de duas variáveis categóricas.

Teste Qui-Quadrado de Independência:
χ² = Σ [(Oij - Eij)² / Eij]

Onde:
• Oij = frequência observada na célula (i,j)
• Eij = frequência esperada sob independência = (Total linha i × Total coluna j) / Total geral

Hipóteses:
• H0: As variáveis são independentes
• H1: As variáveis são associadas

Decisão: Rejeitar H0 se p-valor < α (geralmente 0,05)

🔸 Medidas de Associação para Dados Categóricos

  • Coeficiente Phi (φ): Para tabelas 2×2; varia de 0 (independência) a 1 (associação perfeita).
  • Coeficiente de Contingência (C): Para tabelas maiores que 2×2; ajustado pelo número de categorias.
  • V de Cramér: Generalização do Phi para tabelas r×c; varia de 0 a 1.

🔹 9.5 Distribuições de Probabilidade

📊 Conceito de Distribuição de Probabilidade

Função que descreve a probabilidade de cada possível resultado de uma variável aleatória. Fundamental para modelagem estatística e inferência.

🔸 9.5.1 Distribuição Binomial

Modela o número de sucessos em n tentativas independentes, cada uma com probabilidade constante p de sucesso.

Fórmula de Probabilidade:
P(X = k) = C(n,k) × pk × (1-p)n-k

Onde:
• C(n,k) = n! / [k!(n-k)!] (combinações)
• k = número de sucessos (0 ≤ k ≤ n)

Média e Variância:
μ = n × p
σ² = n × p × (1-p)

Condições de Aplicação:
1. Número fixo de tentativas (n)
2. Cada tentativa tem dois resultados: sucesso/fracasso
3. Probabilidade de sucesso (p) constante
4. Tentativas independentes

🔸 9.5.2 Distribuição Normal

Distribuição contínua simétrica em forma de sino, fundamental na estatística devido ao Teorema Central do Limite.

Função Densidade de Probabilidade:
f(x) = [1 / (σ√(2π))] × e-(x-μ)²/(2σ²)

Propriedades:
• Simétrica em torno da média (μ)
• Média = Mediana = Moda
• Definida por dois parâmetros: μ (localização) e σ (dispersão)

Regra Empírica (68-95-99.7):
• ~68% dos dados estão em μ ± 1σ
• ~95% dos dados estão em μ ± 2σ
• ~99.7% dos dados estão em μ ± 3σ

🔸 Padronização (Escore Z)

Transforma qualquer distribuição normal em uma normal padrão (μ=0, σ=1), permitindo uso de tabelas ou funções padronizadas.

Escore Z:
z = (x - μ) / σ

🔹 9.6 Noções de Inferência Estatística

📊 Conceito de Inferência Estatística

Processo de tirar conclusões sobre uma população com base em informações de uma amostra, quantificando a incerteza por meio de probabilidades.

🔸 9.6.1 Estimação de Parâmetros

🔹 Estimação Pontual

Usa um único valor da amostra para estimar um parâmetro populacional.

  • Média amostral (x̄) estima a média populacional (μ)
  • Proporção amostral (p̂) estima a proporção populacional (p)
  • Variância amostral (s²) estima a variância populacional (σ²)
🔹 Estimação por Intervalo

Fornece um intervalo de valores plausíveis para o parâmetro, com um nível de confiança associado.

Intervalo de Confiança para a Média (σ conhecido):
IC(μ) = x̄ ± zα/2 × (σ / √n)

Intervalo de Confiança para a Média (σ desconhecido):
IC(μ) = x̄ ± tα/2, n-1 × (s / √n)

Intervalo de Confiança para Proporção:
IC(p) = p̂ ± zα/2 × √[p̂(1-p̂) / n]

Nível de Confiança (1-α):
• 90% → z = 1,645
• 95% → z = 1,96
• 99% → z = 2,576

🔸 9.6.2 Testes de Hipóteses

Procedimento para decidir entre duas hipóteses concorrentes sobre um parâmetro populacional, com base em evidência amostral.

🔹 Estrutura de um Teste
  1. Definir hipóteses:
    • H0 (hipótese nula): Afirmação de "nenhum efeito" ou "igualdade"
    • H1 (hipótese alternativa): Afirmação que se deseja evidenciar
  2. Escolher nível de significância (α): Probabilidade máxima de rejeitar H0 quando ela é verdadeira (erro Tipo I). Valores comuns: 0,05, 0,01.
  3. Calcular estatística de teste: Valor padronizado que mede o afastamento da amostra em relação a H0.
  4. Decidir:
    • Rejeitar H0 se p-valor < α
    • Não rejeitar H0 se p-valor ≥ α
  5. Interpretar no contexto: Traduzir a decisão estatística para implicações práticas.

🔸 9.6.4 Testes Paramétricos: Médias e Proporções

🔹 Teste t para uma Amostra

Testa se a média populacional é igual a um valor especificado (μ0).

Estatística de teste:
t = (x̄ - μ0) / (s / √n)

Pressupostos:
• Amostra aleatória
• Dados aproximadamente normais (ou n grande, pelo TCL)
• Observações independentes
🔹 Teste t para Duas Amostras Independentes

Testa se as médias de duas populações são iguais.

Estatística de teste (variâncias iguais):
t = (x̄1 - x̄2) / [sp × √(1/n1 + 1/n2)]

Onde sp = desvio-padrão combinado

Pressupostos adicionais:
• Variâncias populacionais iguais (testar com F ou Levene)
• Amostras independentes
🔹 Teste Z para Proporção

Testa se a proporção populacional é igual a um valor especificado (p0).

Estatística de teste:
z = (p̂ - p0) / √[p0(1-p0) / n]

Pressupostos:
• Amostra aleatória
• np0 ≥ 5 e n(1-p0) ≥ 5 (aproximação normal válida)
✅ Exemplo Prático CESPE: Teste de Hipótese para Média

Cenário: Auditoria quer testar se o valor médio de empenhos do órgão X é diferente de R$ 50.000.

Dados: Amostra de n=36 empenhos; x̄ = R$ 52.500; s = R$ 9.000; α = 0,05.

Passos:
1. H0: μ = 50.000 vs. H1: μ ≠ 50.000 (teste bilateral)
2. Estatística: t = (52.500 - 50.000) / (9.000 / √36) = 2.500 / 1.500 = 1,667
3. Graus de liberdade: df = 35; valor crítico t0,025,35 ≈ 2,030
4. Decisão: |1,667| < 2,030 → Não rejeitar H0
5. Conclusão: Não há evidência estatística, ao nível de 5%, de que o valor médio difira de R$ 50.000.

Observação: O p-valor pode ser calculado: p ≈ 0,105 > 0,05 → mesma conclusão.

⚖️ PADRÕES DE COBRANÇA DO CESPE EM ANÁLISE DE DADOS E ESTATÍSTICA

Padrão 1: Interpretação conceitual sobre cálculo
Tese: O CESPE raramente cobra cálculos manuais complexos. Prefere questões que testam a compreensão conceitual: "O que significa um CV de 30%?", "Quando usar mediana em vez de média?", "Qual a diferença entre análise preditiva e prescritiva?".
Aplicação: Foque em entender o "porquê" e o "quando", não apenas o "como calcular".
Padrão 2: Armadilhas de interpretação estatística
Tese: Questões frequentemente exploram erros comuns: confundir correlação com causalidade, interpretar mal intervalos de confiança ("95% de confiança" ≠ "95% de chance de conter o parâmetro"), ou assumir que outliers devem sempre ser removidos.
Aplicação: Estude os vieses e limitações de cada técnica, não apenas suas fórmulas.
Padrão 3: Aplicação prática em contexto de controle externo
Tese: Muitas questões contextualizam conceitos em situações de auditoria: "Como usar tabela dinâmica para identificar despesas atípicas?", "Qual análise estatística é mais adequada para comparar gastos entre órgãos?".
Aplicação: Relacione cada conceito estatístico a um caso prático de fiscalização do TCDF.
Padrão 4: Excel como ferramenta, não como fim
Tese: O CESPE cobra o uso do Excel para análise de dados, mas sempre com foco na aplicação para tomada de decisão, não em decorar atalhos ou sintaxe de VBA.
Aplicação: Entenda o fluxo: importar dados (Power Query) → transformar → analisar (tabelas dinâmicas, fórmulas) → visualizar (gráficos) → decidir.
Padrão 5: Inferência com amostras
Tese: Questões sobre testes de hipóteses e intervalos de confiança frequentemente testam a interpretação correta dos resultados, não o cálculo manual da estatística de teste.
Aplicação: Domine a lógica: "Se p-valor < α, rejeito H0"; "Um IC de 95% significa que, em 95% das amostras, o intervalo conteria o parâmetro verdadeiro".

📝 QUESTÕES CESPE COMENTADAS

(CESPE/CEBRASPE – TCDF – Analista Administrativo – 2023)
Julgue: Na hierarquia DIKW, a inteligência corresponde ao nível mais elevado, caracterizando-se pela capacidade de prever tendências e orientar decisões estratégicas com base na integração de conhecimento e contexto organizacional.

Comentário: CORRETO. Essa é a definição precisa de inteligência no modelo DIKW: vai além do conhecimento operacional, incorporando visão estratégica, capacidade preditiva e geração de valor organizacional. É o nível que transforma insights em ação estratégica.

(CESPE – SEFAZ DF – 2022)
Julgue: A análise descritiva de dados tem como objetivo principal identificar padrões complexos e prever eventos futuros, utilizando técnicas de machine learning e modelagem preditiva.

Comentário: ERRADO. A questão descreve a análise preditiva, não a descritiva. A análise descritiva responde "o que aconteceu?" e utiliza técnicas como estatística descritiva, agregações e visualizações básicas. A preditiva é que foca em prever o futuro com técnicas avançadas.

(CESPE – CGDF – 2023)
Julgue: Em uma distribuição assimétrica à direita, a média tende a ser maior que a mediana, pois os valores extremos à direita "puxam" a média para cima, enquanto a mediana é mais robusta a outliers.

Comentário: CORRETO. Essa é uma propriedade fundamental das medidas de posição: em assimetria positiva (cauda à direita), a média > mediana > moda. A mediana é menos sensível a valores extremos, sendo preferível em distribuições assimétricas.

(CESPE – TCDF – 2024)
Julgue: Um intervalo de confiança de 95% para a média de despesas com diárias significa que há 95% de probabilidade de que a média populacional esteja contida no intervalo calculado a partir da amostra.

Comentário: ERRADO. Essa é uma interpretação incorreta comum. O nível de confiança refere-se ao procedimento, não ao intervalo específico: "Se repetíssemos a amostragem muitas vezes, 95% dos intervalos construídos conteriam a média populacional verdadeira". O parâmetro é fixo; o intervalo é que varia entre amostras.

(CESPE – SEDES DF – 2023)
Julgue: No Power Query do Excel, a operação "Unpivot" transforma colunas em linhas, convertendo dados no formato "wide" para o formato "long", o que é essencial para análise em tabelas dinâmicas e modelagem estatística.

Comentário: CORRETO. Essa é a definição exata da operação Unpivot (também chamada de "derreter" ou "tidy"). É fundamental para preparar dados para análise, pois a maioria das ferramentas estatísticas e de BI exige dados no formato longo (uma observação por linha).
